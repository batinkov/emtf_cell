HAL (Hardware Access Library)

A library developed for the CMS data aquisition system at CERN

Version 4_0_0-1 10th-April-2007

A high level interface which allows for user friendly hardware access to VME or PCI modules.

The library contains a software layer between the user program and the driver software to access hardware. A class called "BusAdapter" implements the interface to the various hardware drivers. Several BusAdapters are contained in the distribution. It is easy for the user to extend this collection by writing his own BusAdapter.

The HAL library currently supports VME and PCI hardware accesses. For the current list of supported BusAdapters please refer to the online HAL homepage.

If you use XML AddressTables you need the schema against which your address table will be checked. It is contained in the root directory of the distribution (HardwareAddressTable-ver-2-0.xsd) or it is available at http://xdaq.web.cern.ch/xdaq/xsd/HardwareAddressTable-ver-2-0.xsd.

It is assumed you use the HAL library in the context of the XDAQ online environment of CMS. There are two dependencies on libraries in the TriDAS directory tree:

  • The xerces package is needed to support XML Addresstables. It should be installed in $XDAQ_ROOT/daq/extern.
  • The exceptions thrown by the HAL are derived from the baseclass in the XDAQ xcept package.
If for some reason the HAL must be installed without the XDAQ environment, these two libraries can be installed independently and the Makefile.in can be edited in order set variables which point to the location of the two needed libraries.


Documentation

  • The Users Guide contains a complete description of the library including architecture, functionality, installation guide, user-instructions and instructions how to extend the library (for example to custom hardware drivers). It is available in pdf format.
  • The API documentation of the current version (see below) of the HAL library can be browsed here. It is the most important document for user's prgramming with HAL. The API of the BusAdapters can be found here. (In the API documentation there are cross links). A small library of useful utilities which you can use is documented : API of the Utilities library
  • Examples The HAL contains a series of examples where the usage of the various components is demonstrated. Among these a small application to measure the overhead due to the HAL library for a single hardware access is included. Also a complete XDAQ application using the HAL library is included.
